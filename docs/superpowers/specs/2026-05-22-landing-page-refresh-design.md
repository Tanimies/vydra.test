# VYDRA CORE Landing Page Refresh Design

## Goal

Refresh the existing VYDRA CORE landing page so it reflects the product as it exists today: a balanced AI learning platform for students and a classroom-ready exam preparation system for educators. The page should keep the current overall structure and warm beige/brown/cream visual language, but present the product with stronger hierarchy, sharper copy, and a more premium sense of completeness.

## Positioning

The landing page should communicate both halves of the product at once:

- for students: study from uploaded material, learn with AI chat, generate quizzes, and track progress
- for educators: run classrooms, publish materials, create quizzes, monitor performance, and intervene quickly

The primary product impression should be:

1. premium
2. practical
3. classroom-capable
4. grounded in real study material

The core message is not "just another AI tutor" and not "just another classroom tool." It is a connected study and assessment platform.

## Approved Hero Direction

### Eyebrow

`VYDRA CORE`

This stays prominent and should be visually larger than before.

### Headline

`Smarter Learning Starts Here.`

Each word should remain visually styled so the first letter stands out more than the rest, but the overall effect must still feel elegant and warm rather than loud.

### Accent Treatment

Use `#B45309` for the highlighted first letters in the landing-page headline. This gives better contrast and pop than the darker brown already used elsewhere, while remaining fully aligned with the beige/cream/brown palette.

### Supporting Copy

The supporting hero paragraph should clearly express the two-sided nature of the product:

- AI-guided learning for students
- classroom-ready workflows for educators

It should mention uploaded material, quizzes, classrooms, and intervention or support without sounding overloaded.

## Page Structure

The page should keep the same broad section rhythm as the current landing page, but each section should be rewritten to match the mature product.

### 1. Hero

Purpose:
- establish VYDRA CORE as a premium AI learning and classroom system
- give immediate student and educator relevance
- keep the existing CTA area

Content emphasis:
- uploaded study material
- AI learning loop
- educator workflows

### 2. Core Value / Product Flow Card

Purpose:
- show how the system works at a glance
- make the balanced student + educator flow obvious

This section should still be compact, but the wording should move beyond generic "one system, two role-specific loops" and instead describe the real operational loop:

- study from material
- practice with quizzes
- track performance
- support or intervene when needed

### 3. Student and Educator Parallel Sections

Purpose:
- preserve the split layout already on the page
- make each side feel more feature-complete and premium

Student mode should foreground:
- uploaded material
- offline PDF study
- learning chat
- adaptive quizzes
- progress tracking

Educator mode should foreground:
- classrooms
- manual and AI quiz creation
- Bloom workflows
- communication and intervention
- proctored assessment

### 4. Capability Section

Purpose:
- present premium differentiators, not generic features

This section should highlight the most marketable strengths:
- study from your own material
- classroom-ready workflows
- proctored quizzes and answer-key grading
- progress and intervention visibility

### 5. Platform Coverage / Trust Section

Purpose:
- reinforce that the product is already broad and functional
- make the app feel real rather than conceptual

This section should describe the platform as a connected system for:
- study
- assessment
- communication
- intervention

## Visual Direction

### Keep

- beige / brown / cream palette
- editorial heading feel
- warm product identity
- current high-level page structure

### Improve

- stronger hero hierarchy
- more premium copy rhythm
- less early-MVP wording
- clearer student/educator balance
- more specific and trustworthy feature presentation

### Avoid

- generic SaaS filler language
- over-explaining technical features
- sounding like a concept pitch instead of a usable product
- introducing a totally different layout that breaks the page familiarity the user wants to keep

## Copy Principles

The copy should:

- sound confident, clear, and product-grade
- stay concise
- emphasize outcomes over taxonomy or internal system language
- use title case selectively where it improves polish
- avoid visible references to "Google Classroom-style" or comparison wording in the website UI

## UI Constraints

- this redesign is only for the website UI
- internal docs can still reference design comparisons if needed
- current navigation and entry points should remain intact
- no backend changes are required

## Verification

The refresh is complete when:

- the landing page hero reflects the updated product scope
- the headline uses `Smarter Learning Starts Here.`
- the headline initials use `#B45309`
- visible frontend copy no longer uses `Google Classroom` comparison phrasing
- the page still builds successfully with the existing Next.js setup
